Developer-facing prototype · Domain-calibrated RHCA

Evaluate AI behavior before you ship.

DeeProject turns multi-turn AI conversations into release-ready behavioral QA reports: RHCA scores, failure evidence, severity, suggested fixes, and domain-specific sub-dimensions that teams can calibrate to their product.

Input
CSV / JSON
Batch test sets, traces, or manual conversations.
Core signal
RHCA
Reasoning, Helpfulness, Consistency, Alignment.
Calibration
Domain
Editable sub-dimensions by product context.
Output
Gate
Approve, review, or block release.
Run Metadata
Track which model, build, and environment this behavioral QA run evaluates.

Developer Integration Examples
Use this section to show engineers how DeeProject can fit into CI, eval jobs, or agent release workflows.
const response = await fetch("/api/rhca/evaluate", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    metadata: {
      project: "EarSketch AI",
      environment: "Staging",
      model: "gemini-rag",
      build: "v0.18",
      test_suite: "coding-debugging-regression"
    },
    calibration: {
      domain: "coding",
      reasoning: ["explains error cause", "connects fix to code behavior"],
      helpfulness: ["provides minimal runnable fix", "uses correct API"],
      consistency: ["does not contradict earlier turns"],
      context_alignment: ["adapts to learner level", "uses user code context"]
    },
    turns: [
      {
        turn_id: 1,
        user: "This doesn't work: fitMedia(sounds, 1, 1, 5)",
        ai: "Use sounds[0] instead of sounds."
      }
    ]
  })
});

const report = await response.json();
